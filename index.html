Welcome to Nodecast server.

Nodecast aimed to be a worker scheduler, also his agnoctic so yours workers can use any kind of langage, your worker must read STDIN to receive a payload and write to STDOUT to push a payload. Your worker must to be managed with NCW which is connected to NCS. Nodecast can push any kind of binary data to your worker or your pool of worker.

Nodecast is using deeply Zeromq to communicate between yours workers and MongoDB to store yours payloads and the workers's status.

FUTUR

Nodecast embeds a XMPP server and soon a torrent tracker. These features will be explains later.

The next big step is to embeds a web interface to manage yours workers, kwow their status, etc ...

TECH

Nodecast is using a full Qt stack (C++) so he has a tiny memory print and he is really quickly to perform yours tasks.

Launch NCS

$ ncs --mongodb-ip=127.0.0.1 --mongodb-base=nodecast_prod --domain-name=localhost --xmpp-client-port=6222 --xmpp-server-port=6269 --smtp-hostname="your.server.mail" --smtp-username="your-user-account" --smtp-password="your-password" --smtp-sender="your-email-sender" --smtp-recipient="your-email-recipient"

Nodecast can use smtp protocol to send you an email alert when a worker is down.
