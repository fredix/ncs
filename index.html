Welcome to Nodecast server.

Nodecast is a worker scheduler, also it's agnoctic so yours workers can use any kind of langage. Workers must read STDIN to receive a payload and write to STDOUT to push a payload. Your worker is managed though NCW (nodecast client) which is connected to NCS. Nodecast can push any kind of binary data to a worker or to a pool of worker.

Nodecast is using deeply Zeromq to communicate between yours workers and mongoDB to store yours payloads and the workers's status.

It embeds an HTTP server to expose an API.

FUTUR

Nodecast have also an XMPP server and soon a torrent tracker. These features will be explains later.

The next big step is to embeds a web interface to manage yours workers, kwow their status, etc ...

TECH

Nodecast is using a full Qt stack (C++) so he has a tiny memory print and he is really quickly to perform yours tasks.

Launch NCS

$ ncs --mongodb-ip=127.0.0.1 --mongodb-base=nodecast_prod --domain-name=localhost --xmpp-client-port=6222 --xmpp-server-port=6269 --smtp-hostname="your.server.mail" --smtp-username="your-user-account" --smtp-password="your-password" --smtp-sender="your-email-sender" --smtp-recipient="your-email-recipient"

Nodecast can use smtp protocol to send an email alert when a worker is down.

USAGE

You need to create a node and a workflow (see NCW page)

push your data

curl -H "X-node-uuid: 2d0a7780-e8fe-4e0a-89c6-a5a2737b095a" -H "X-node-password: 2d0a7780-e8fe-4e0a-89c6-a5a2737b095a" -H "X-workflow-uuid: 0ebcdab6-0263-42d3-be7d-9602fa15f68c" -H "X-payloadfilename: filename" -X POST --data-binary @filename http://127.0.0.1:4567/payload/create

return a session uuid : 

{"uuid":"7b32b9d0-3ea9-4663-9577-4b34192055bf"}

Nodecast will store your data to GridFS and push it to your first worker according your workflow.
